**本人であること**を確実に確かめることでセキュリティを高めることが目的であった**認証**に対し、

## Microsoft Defender XDRについて

### Microsoft Defender XDRの概要

Microsoft DefenderXDRとは、**Microsoft 365での脅威保護を担う製品群**です。  
エンドポイントやID、Office 265サービスなど、その保護対象は多岐にわたります。  

代表的な製品としては以下のようなものがあげられます。  
- Microsoft Defender for Endpoint
- Microsoft Defender for Office 365
- Microsoft Defender for Identity

他にも機能としてはありますが、今回は代表的なものをピックアップしています。  
以下のリンクでMicrosoft Defenderの製品とサービスについて説明されていますので、詳細はこちらをご覧ください。  
[Microsoft Defender の製品とサービス](https://learn.microsoft.com/ja-jp/defender/)

### Microsoft Defender for Endpoint

Microsoft Defender for Endpointは、  
**エンドポイント（PC、サーバーなど）をサイバー攻撃から保護する**製品です。  

サイバー攻撃の主流な対策はアンチウィルスからエンドポイント保護へと変わってきています。  
アンチウィルスが「侵入されない」を目的としていたのに対し、  
エンドポイント保護では、「（侵入されても）広げない」を目的としています。  
このような変化の背景には**サイバー攻撃の高度化**と**ゼロトラストの浸透**があげられます。  
この二つについても、サイバー攻撃はもはや防ぎきることが難しくなったので、  
何も信頼せずすべて検証しましょうね（＝ゼロトラスト）、となったわけなので関連する事象ではあります。  

主な機能としては以下のようなものがあげられます。  
- エンドポイントでの検出と対応
- 調査と修復の自動化
- セキュリティ状態の評価

これらの機能を組み合わせることで、組織のセキュリティ対策の**門番**として攻撃面の縮小と、  
最新情報を活用した速やかな対応が可能となっています。  

### Microsoft Defender for Office 365

Microsoft Defender for Office 365は、  
**Microsoft 365環境を脅威から保護する**製品です。  

ユーザー側に分かりやすいところで言うと、  
添付ファイルのスキャンであったり、リンクの検査であったりと、  
メールによるセキュリティ保護をイメージしておけばいいのではないかな、と思います。  

ということで、主な機能は以下の通りです。  
- 添付ファイル保護・安全なリンク
- フィッシング対策
- リアルタイム検出

そのほか、上位プランでは訓練機能や高度な調査機能が提供されています。  
エンドポイントと同じく個人のリテラシーで防ぎきることは困難になってきていますので、  
このようなソリューションで技術的にも防御していくことが大切です。

### Microsoft Defender for Identity

Microsoft Defender for Identityは、  
**IDへの脅威を検知・調査・自動修復を提供する**製品です。  

機能としては以下のようなものがあげられます。  
- IDのセキュリティ体制評価
- 環境全体の脅威の検出
- 疑わしいアクティビティの特定
- アラートとアクティビティの調査

特にID管理はクラウドとオンプレにまたがって運用されていることもあります。  
オンプレ側の情報もクラウド側に取得し、分析することで攻撃の被害を減らすことが可能となります。

## M365における情報保護について

### M365における情報保護の概要

Microsoft 365での情報保護はAzure Information Protectionによって実現されます。  

目的は**組織の機密情報・重要データの意図しない漏洩や持ち出しを防ぐこと**です。  

### Azure Information Protection

Azure Information Protectionは、  
**機密情報に秘密度ラベルを付け、暗号化とアクセス制御を適用するサービス**のことです。  

このような技術は**DLP（Data Loss Prevention）**と呼ばれ、  
ゼロトラストと同じようにクラウド全盛の昨今、重要な考え方とされています。  

ラベルは「社外秘」「公開」などで定義され、分類する対象はMicrosoft 365テナント上の情報資産となります。  
アクセス制御はユーザー・グループ単位、あるいは組織内外で適用する必要があります。   
自動ラベルポリシーの設定も可能なので、ユーザー側の負担を減らすこともできます。
