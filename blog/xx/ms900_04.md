## Microsoft Defender XDRについて

### Microsoft Defender XDRの概要

Microsoft DefenderXDRとは、**Microsoft 365 環境における脅威保護を担う製品群の総称**です。  
エンドポイントやID、Microsoft 365サービスなど、その保護対象は多岐にわたります。  

代表的な製品としては以下のようなものがあげられます。  
- Microsoft Defender for Endpoint
- Microsoft Defender for Office 365
- Microsoft Defender for Identity

他にも関連する製品はありますが、今回はMS-900で押さえておきたい代表例をピックアップしています。  
製品全体像についてはMicrosoft公式ドキュメントがわかりやすいので、詳細はこちらをご覧ください。  
[Microsoft Defender の製品とサービス](https://learn.microsoft.com/ja-jp/defender/)

### Microsoft Defender for Endpoint

Microsoft Defender for Endpointは、  
**エンドポイント（PC、サーバーなど）をサイバー攻撃から保護する**製品です。  

従来の主流だったアンチウィルス対策は「侵入されないこと」を目的としていました。  
一方、エンドポイント保護では、「（侵入されても）広げないこと」を重視しています。  
この考え方の背景には**サイバー攻撃の高度化**と**ゼロトラストの浸透**があります。  
サイバー攻撃はもはや完全に防ぎきることが難しくなったので、  
「何も信頼せず、すべて検証する（＝ゼロトラスト）」という考え方が重要になってきたというわけですね。  

主な機能は以下のとおりです。  
- エンドポイントでの検出と対応
- 調査と修復の自動化
- セキュリティ状態の評価

これらを組み合わせることで、組織のセキュリティ対策の**門番**として攻撃面の縮小と、  
最新の脅威情報を活用した速やかな対応が可能になります。  

### Microsoft Defender for Office 365

Microsoft Defender for Office 365は、  
**Microsoft 365環境を脅威から保護する**製品です。  
（特にメールやコラボレーションサービス）

ユーザー視点で分かりやすい例としては、  
**添付ファイルのスキャン**や、**リンク（URL）の検査**といった、  
メール周りのセキュリティ対策をイメージしておけばよいと思います。  

主な機能は以下のとおりです。  
- 添付ファイル保護・安全なリンク
- フィッシング対策
- リアルタイム検出

上位プランでは疑似攻撃メールを用いた訓練機能やより高度な調査機能が提供されています。  
エンドポイント対策と同様に、個人のリテラシーで防ぎきることは困難になってきているため、  
技術的な対策を組み合わせることが重要です。

### Microsoft Defender for Identity

Microsoft Defender for Identityは、  
**IDへの脅威を検知・調査・自動修復を提供する**製品です。  

機能としては以下のようなものがあげられます。  
- IDのセキュリティ体制評価
- 環境全体の脅威の検出
- 疑わしいアクティビティの特定
- アラートとアクティビティの調査

ID管理はクラウドとオンプレADにまたがって運用されていることも少なくありません。  
オンプレ側の情報も含めて可視化・分析することで攻撃の被害を減らすことが可能となります。  

### その他のDefender製品

他にも**クラウドアプリの保護**や**脆弱性の管理**、**IoTデバイスの保護**など多くの製品が提供されています。  
本当はもう少し触れておきたいこともありますが、  
MS-900全体のまとめということでこのくらいにとどめておきます。  

## M365における情報保護について

### M365における情報保護の概要

Microsoft 365での情報保護はMicrosoft Purview Information Protectionによって実現されます。  
（過去にはAzure Information Protectionと呼ばれていたこともあります。）

目的は**組織の機密情報・重要データの意図しない漏洩や持ち出しを防ぐこと**です。  

### Microsoft Purview Information Protection

Microsoft Purview Information Protectionでは、  
**機密情報に秘密度ラベルを付け、暗号化とアクセス制御を適用**しています。  

このような考え方は**DLP（Data Loss Prevention）**と呼ばれ、  
ゼロトラストと同様クラウド時代には欠かせない概念とされています。  

ラベルは「社外秘」「公開」などで定義され、Microsoft 365テナント上の情報資産に適用されます。  
アクセス制御はユーザー・グループ単位、あるいは組織内外といった条件で設定可能です。   
また、自動ラベルポリシーを利用してユーザー側の負担を減らすこともできます。
