こんにちは、Bindです。  

ここまでいろいろと投稿をしてきて、  
「そういえば一度も画像を貼っていないな」ということを思いました。  

遊んだゲームのプレイ記事も書いていますし、この先の投稿にも一切画像がないのは寂しい！  
ということで今回は、**Markdownで画像を表示する方法**をまとめてみようと思います。  

## 仕組み

既存のブログツールを使っていると、  
テキストボックスの中に画像をドラッグ＆ドロップすれば、投稿内に反映されますよね。  
ただ、実際にはテキスト画像が一体となって保存されているわけではありません。  

文章の中で表示したい画像を指定して呼び出している、という仕組みになっています。  
つまり、**画像の実体はテキストと別のところに存在している**ということになりますね。  

## 作業

### 画像の置き場所の作成

まず記事の中に表示をする画像の置き場所を作成します。  
今回はこんな感じで管理しましょう。  

【フォルダ構成】
```
images/
→記事ごとのフォルダ/
```

……、すでに管理が面倒になっていくような気がしています。  
普段は便利なツールに任せている分、  
「本来はこういう構造になっているんだな」という理解にはなりますけども……。  

しばらく運用してみて、改善できそうなら都度見なおしていこうと思います。  

### 画像の参照

置き場を準備したら、次はそれを記事から参照します。  

まずHTMLで記載するとこんな感じです。  

【HTML】
```
<img src="/images/記事ごとのフォルダ/表示する画像"
     alt="画像の説明文">
```

そして、これをMarkdownで書くとこんな感じです。  

【Markdown】
```
![画像の説明文](/images/記事ごとのフォルダ/表示する画像)
```

HTMLと比べるとシンプルですね、やっぱり早めに実装してよかったなと思います。  

……、ということで。  
特に画像の方を準備していなかったので取り急ぎ準備しました。  
マウスでSairyHubのロゴを書いてみました。  
![SairyHubロゴ 案](/img/image_01/SairyHub_logo.png)  

高校生くらいの時はこういうカリグラフィーぽい文字を書くのが好きだった気がします。  
そのうちちゃんとしたロゴにしたいですね、今回はとりあえず原案ということで。  

## 最後に

今まで記事を書きながら「画像を投稿したいなあ」と思いつつ後回しにしていました。  
ただ、今回は先にMarkdown化していたのが功を奏したようで、思ったよりもスムーズに対応できました。  
これで今後はもう少し読みやすい記事が書けそうです。  

今のところ全く触っていません（ChatGPTさんに作ってもらってます）が、  
そのうち触ることになるであろうCSSにボコボコにされる未来が見えます。  
ただでさえデザインとか苦手なのに……。  
とはいえ、誰に頼まれているわけでもないサイト運営にグダグダ文句言っても仕方ないですね。  
ロードマップは長いので、これからも頑張っていきます。  

それでは、ここまで読んでいただきありがとうございました。
