## 現運用のおさらい

もともとのブログ運用は、
**1記事＝1HTMLファイル**という、かなり素直な構成でした。  

- 記事ごとに HTML を作成
- ヘッダー・フッターは共通化してインクルード
- 本文・タイトル・日付・カテゴリは各 HTML に直接記述

HTML や CSS の勉強も兼ねていたので、この形自体に大きな不満はありませんでした。
ただ、記事が増えてくるにつれて、少しずつ違和感も出てきました。

文章を書くたびにタグを書き、
ちょっとした修正でも HTML ファイルを開き、
結果として「書く」より「整える」に時間を取られている感じです。

このままだと、軽いメモや思いつきを残すのが億劫になりそうだな、
というのが方針転換のきっかけでした。

## 実際にした作業
### 1. 記事本文を Markdown に切り出す

最初にやったのは、とにかく 本文を HTML から追い出すことでした。

- 記事本文を .md ファイルとして作成
- 見出しやリストは Markdown 記法で記述
- HTML タグは極力使わない

これだけでも、文章を書く心理的ハードルがかなり下がりました。
「構造を考えず、とりあえず書ける」というのは大きいですね。

### 2. 表示用の HTML を 1 枚に集約

次に、記事表示用の HTML を 1 枚だけ用意しました。

- ヘッダー・フッターは従来通り共通化
- 記事タイトル・日付・カテゴリの表示枠を HTML 側に固定
- 本文だけを JavaScript で差し替え

JavaScript で Markdown ファイルを読み込み、
HTML に変換して表示する形にしています。

これで、「記事ごとの差分」は Markdown だけになりました。

### 3. Markdown にメタデータを持たせる

最後に、Markdown 側にメタデータを持たせました。

- タイトル
- 日付
- カテゴリ

いわゆる Front Matter ですね。
これを JavaScript で読み取って、HTML の表示部分に反映しています。

結果として、
記事に関する情報はすべて Markdown に集約され、
HTML は「表示の器」としての役割に専念する形になりました。

### 総まとめ

最終的な構成は、こんな感じになりました。

- 記事本文とメタ情報は Markdown
- 表示レイアウトは HTML 1 枚
- 切り替えや読み込みは JavaScript

HTML を量産しなくてよくなり、
「書くこと」と「見せること」がきれいに分離できたのは大きな収穫です。

正直、最初は
「静的サイトでここまでやる必要ある？」
と思う部分もありました。

でも、仕組みを一度作ってしまえば、
あとは Markdown を追加するだけで記事が増えていきます。

勉強にもなりましたし、
これからブログを続けていくための土台としては、
なかなか悪くない選択だったと思っています。

次は、この構成を前提に
一覧ページやカテゴリ表示も自動化していきたいですね。
