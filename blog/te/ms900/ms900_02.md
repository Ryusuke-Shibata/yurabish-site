## 生産性ソリューションについて

### 生産性ソリューションの概要

Microsoft 365における生産性ソリューションとは、主に以下のサービスを指します。  
- Word
- Excel
- PowerPoint
- Outlook

いわゆるOfficeアプリケーションだと思っていただければ問題ないかと思います。  
それぞれの使い方の詳細については、本記事では割愛します。  
MS-900 の試験範囲でも、アプリ操作そのものは問われません。  
（こんなことを言っているので僕はいつまで経ってもこれらのアプリをうまく使えていません……。）  

### Microsoft365における提供形態

Microsoft 365ではこれらのサービスが**Web版**で、プランによっては**アプリケーション版**で提供されます。  
アプリケーション版は「Microsoft 365 Apps」という形でアドオン提供もされています。  
Web 版についてはモバイル端末からも利用できます。  

働き方が多様化する中で、**場所やデバイスを選ばずに利用できる**という点は、生産性向上という意味でも大きなメリットですね。  （そういいつつもモバイル版の使い勝手は個人的にはいまいちです。  
　外出先でモバイルからファイルの確認と軽微な修正ができればそれで十分だとは思いますが。）  

### オンプレミス版との違い

Officeアプリケーションと大体同じ、と表現しましたがもちろん違いはあります。  
| 項目 | クラウド版 | オンプレミス版 |
|----- | ----- | ----- |
| 費用 | サブスクリプション | 買い切り ❘
| バージョン | 常に最新バージョン | インストール時のバージョン |
| 課金単位 ❘ ユーザーごと | デバイスごと |
| 共同編集 | 可能 | 不可能 |

特に重要なのが**課金単位の違い**です。  
クラウド版は**ユーザごとにアプリケーションを利用する権利**が付与されます。  
例えばAさんに該当のライセンスが割り当てられるとAさんがサインインする複数のデバイスでアプリケーションが利用可能となります。  
一方オンプレミス版はデバイス単位の課金となるので、Aさんが2台の端末を利用する場合はライセンスも2式必要となるわけです。

また、クラウド版は後述する**コラボレーションソリューションとの連携**に前提となっており、  
これこそがMicrosoft 365の大きな特徴にもなっています。

### その他の生産性ソリューションについて

ここまでは有名どころを挙げていきましたが、このほかにも生産性向上を支援するさまざまなサービスが用意されています。  
- Microsoft Planner
- Microsoft Bookings
- Microsoft To Do
- Microsoft Forms
- Microsoft Lists
- Microsoft Loop
- などなど

これらは資料作成よりもタスク管理や情報整理を支援するためのツールです。  
用途に応じて組み合わせて使うことで、業務効率を高めることができます。  

## コラボレーションソリューションについて

### コラボレーションソリューションの概要

Microsoft 365を利用するにあたってはこのコラボレーション領域が大本命です。  
中核となるMicrosoft Teamsを中心に、組織内外のメンバーとのコミュニケーションや共同作業を、  
スムーズに行うための機能が提供されています。  
適切なセキュリティ設定は必要ですが、社内に限らず外部とのやり取りにも活用できる点は、  
クラウドサービスならではの利点と言えるでしょう。  

### Microsoft Teams

まずは有名どころであろうMicrosoft Teamsについてまとめていきましょう。  
こちらは単なるチャットツールではなくMicrosoft 365におけるコラボレーションのハブの役割を担うサービスです。  
主な機能としては、以下のようなものがあります。  
- チャット・チームでのコミュニケーション
- 会議（音声・ビデオ・画面共有）
- ファイル共有（SharePoint と連携）
- 他アプリ（Planner / OneNote / Forms など）との統合

突き詰めていけばMicrosoft 365のほぼすべての機能がMicrosoft Teams上で利用できます。  
毎朝大量のウィンドウを開いて作業しなくても済む、というのが本来の理想形ですね。  
（僕は相変わらずウィンドウを散らかしていますが……。） 

### OneDrive・SharePoint

OneDriveとSharePointは、どちらもストレージ機能を持つサービスです。  
ただし、役割には明確な違いがあります。  
- OneDrive：**個人用ストレージ**
- SharePoint：**チーム・組織用ストレージ**

Teamsを軸に考えると、違いが分かりやすくなります。  
Teams でファイルを共有する際、実際にはファイルが直接メッセージに添付されるわけではありません。  
クラウド上にファイルが保存され、**そのリンクが共有される仕組み**です。  
- 個人チャットで共有されたファイル → OneDrive
- チームで共有されたファイル → SharePoint

「ファイルの実体はどこに保存されているのか」を意識すると、  
OneDriveとSharePointの違いは理解しやすくなります。 

### 共同編集

Teams、OneDrive、SharePointが連携することで実現される代表的な機能が**共同編集**です。  
OneDrive・Sharepoint上のWordなどのファイルを複数人で同時に編集することが可能になります。  
共同編集ができないオンプレミス版であれば、順番に作業を完了させてから次の人が作業をし、  
うまく回らなければ版が複数に分岐してファイルだけが増えていき、どれが最新なのかがわからない、なんてことも……。  
組織での作業の生産性向上のためには、この共同編集の活用が不可欠ではないかな、と思います。
