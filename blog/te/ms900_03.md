## Microsoft Entra IDについて

### Microsoft Entra IDの概要

Microsoft Entra IDとは、Microsoft 365におけるセキュリティ機能の中核となるサービスです。  
役割としては**Microsoft 365の認証基盤**となります。  
つまり、ユーザ・グループ・デバイスなどの組織のリソースの管理が主な機能となる、  
**クラウドベースのID管理サービス**です。

### Microsoft Entra IDによって提供される機能

主な機能としては以下の3つとなります。
- ユーザ認証
- シングルサインオン
- オンプレADとの連携

ユーザ認証はのちの項目で掘り下げますので、この項目では**シングルサインオン**と**オンプレADとの連携**についてまとめておきましょう。  

まず、シングルサインオンについてです。  
**SSO**と表記されることが多く、さまざまなクラウドサービスを利用することになった昨今では、  
導入するメリットが大きくなってきた機能ではないかと思います。  
どのような機能かというと、「1つのID・パスワードを利用して複数のクラウドサービスへのアクセスができるようになる機能」です。  

次に、オンプレADとの連携についてです。  
こちらは**Microsoft Entra Connect**という機能を利用します。  
Microsoft Entra IDは**クラウドサービス向けの認証基盤**です。  
オンプレ製品への認証基盤が必要な場合は**Active Directory**というサービスを利用することとなります。  
いずれも「Microsoftが提供する認証基盤」という共通点があります。  
そういった経緯から（？）、Microsoft Entra IDもかつては「Azure Active Directory」と呼ばれていましたが、  
役割も違い、混同しやすいのでMicrosoft Entra IDという呼称に改められたのかな、と思ってます。 
ということなので、「オンプレAD環境がある＝Entra ID Connectをする」**というわけではありません**。  
あくまで利便性の向上を目的として導入するものですので、  
組織の規模を鑑み、導入しないというのも一つの選択でしょう。  

## MFAと条件付きアクセスについて

### ユーザ認証機能の概要

Microsoft Entra IDでは**IDを守る策の一つ**としてユーザをどのように認証するかを選択する機能があります。  
その代表的なものがMFAと条件付きアクセスです。  
例えば、組織外の人間に組織の人間のIDが侵害されてしまうと組織内のデータにアクセスされてしまいます。    
また、組織内の人間であっても、適切な時間・場所以外（業務外の自宅など）から組織内のデータにアクセスできるのも、  
セキュリティ的にはいい状態とは言えません。  
このように、**適切なユーザなのか**を認証するための機能がMicrosoft Entra IDには備わっています。

### MFA

MFA（Multi Factor Authentication）は多要素認証のことです。  
よくある認証としてはIDに対しパスワードで認証する1要素の認証ですが、  
MFAでは以下の3要素を組み合わせて本人確認を行います。  
- 知識情報：パスワード・PINなど
- 所持情報：デバイス・トークンなど
- 生体情報：指紋・顔など

よくあるMFAの実装としては、知識情報としてパスワードを入力した後、  
所持情報の確認のため、スマホに送られてきたワンタイムパスワードを入力するケースや、  
生体情報の確認のため、指紋認証を実施するケースなどがあげられます。  
これらによって、間違いなく本人であることが確認できます。  

ただ、その一方で、**適切なアクセスであるかどうか**は確認できません。  
その部分は次の**条件付きアクセス**でカバーすることになります。

### 条件付きアクセス

条件付きアクセスとは、以下のような条件を組み合わせてアクセスの許可・拒否を制御する仕組みです。  
- 誰が（ユーザー）
- どこから（IPアドレス・位置情報）
- 何で（デバイス・アプリ）

先述の通り、社外や管理外のデバイスからのアクセスは禁止してしまうこともできます。  
反対に、利便性向上を目的として、社内からのアクセスの場合はMFAを強制せず、  
社外からのアクセスにのみMFAを必須とすることもできます。  

また、ここまでは**接続元**に関する話でしたが、  
重要性によって**接続先**の制限も可能です。
